## 🔴 端云协同架构（v3.6 — 评委明确指引）

### "端云协同"的官方立场

评委原话：**"端侧大模型能力有限，最好是端云协同"**。这推翻了 v3.1-v3.5 的"纯端侧"假设。

正确架构：

```
用户发消息 "奶奶，给我讲个你年轻时候的故事吧"
        │
        ▼
┌─────────────────────────────────────┐
│       智能路由层（chat.js）           │
│                                     │
│  简单对话？  → 端侧 BlueLM 3B        │
│  隐私敏感？  → 端侧 BlueLM 3B        │
│  复杂故事？  → 云端 BlueLM 70B       │
│  情感回忆？  → 云端 BlueLM 70B       │
│  快速关心？  → 端侧 BlueLM 3B        │
└─────────────────────────────────────┘
        │                    │
        ▼                    ▼
┌───────────────┐    ┌──────────────────┐
│ 端侧 (NPU)     │    │ 云端 (vivo API)   │
│ BlueLM 3B      │    │ BlueLM 70B       │
│ MTK DX5 NPU    │    │ HTTPS API        │
│ 离线/低延迟     │    │ 高质量/复杂推理    │
│ 隐私数据不离开  │    │ 故事/情感/上下文   │
└───────────────┘    └──────────────────┘
```

### 端云分工策略

| 对话类型 | 路由 | 引擎 | 理由 |
|----------|------|------|------|
| 日常关心（"吃了吗""注意身体"） | **端侧 3B** | LlmManager NPU | 简单模式匹配，端侧够用，隐私安全 |
| 快速叮嘱（"多穿点"） | **端侧 3B** | LlmManager NPU | 低延迟(<0.2s首字)，离线可用 |
| 隐私话题 | **端侧 3B** | LlmManager NPU | 数据不出设备 |
| 长故事讲述 | **云端 70B** | 蓝心 API | 需要强上下文理解和创造力 |
| 情感回忆 | **云端 70B** | 蓝心 API | 需要细腻情感表达 |
| 复杂多轮对话 | **云端 70B** | 蓝心 API | 128K上下文窗口 |
| TTS 语音合成 | **云端** | vivo AI 语音 | 端侧3B无TTS能力 |

### 部署形态双轨

```
形态A：Android APK（推荐）
  WebView 壳 → 加载 H5 页面 → JavascriptInterface 桥接
  → 端侧: LlmManager.generate() 调用 NPU
  → 云端: fetch() 调用 Flask /api/chat（后端调蓝心API）

形态B：Web PWA（备选，纯浏览器）
  → 端侧: 不可用（浏览器无法访问NPU）
  → 云端: fetch() 调用 Flask /api/chat（后端调蓝心API）
  → PPT 说明: "纯浏览器端暂不支持端侧推理，APK版本已接入"
```

### 步骤 0.0：创建专用 Python 3.10 环境（最关键的一步）

**推荐 Python 3.10**。理由：
- 所有依赖（Flask、edge-tts、aiohttp）都有预编译 wheel，安装零报错
- Python 3.11/3.12 部分库缺 wheel，3.13 兼容问题更多
- 3.10 是当前 Flask 生态兼容性最广的版本，也是最稳的版本

你的 base 环境有 PyTorch 等 ML 库，不要往里装 Web 开发的包。**新建一个干净的 conda 环境，只放 Python 3.10。**

---

### C. 关键 API 速查

| 方法 | 路径 | 功能 | 关键参数 |
|------|------|------|----------|
| POST | /api/voices | 创建声音档案 | `{"name":"奶奶"}` |
| GET | /api/voices | 获取所有声音 | - |
| GET | /api/voices/{id} | 获取单个声音 | - |
| DELETE | /api/voices/{id} | 删除声音 | - |
| POST | /api/voices/{id}/upload | 上传音频 | FormData `audio_file` |
| GET | /api/voices/{id}/model-status | 查询训练进度 | - |
| POST | /api/chat | **核心：AI对话** | `{"voice_id","message","history"}` |
| GET | /api/voices/{id}/conversations | 获取对话列表 | - |
| GET | /api/conversations/{id} | 获取单次对话 | - |
| GET | /api/stats | 统计数据 | - |
| POST | /api/voices/{id}/suggest-topics | 话题推荐 | `{"context":"..."}` |
| GET | /api/config | 系统配置信息 | - |

### D. 每日工作流程

```
1. 打开 VS Code → 打开 d:\echo-app 文件夹
2. 打开 Anaconda Prompt → cd d:\echo-app → conda activate echo-app → python app.py
3. 手机浏览器打开 http://你的IP:5000
4. 按当天阶段粘贴 Prompt 给 Claude Code
5. AI 生成代码后 → 浏览器刷新验证 → 手机验证
6. 有问题 → 🔧修复模板 → AI修复 → 再验证
7. 验证通过 → 在验收清单该项打勾
8. 当日完成 → 在计划中标记进展
```
